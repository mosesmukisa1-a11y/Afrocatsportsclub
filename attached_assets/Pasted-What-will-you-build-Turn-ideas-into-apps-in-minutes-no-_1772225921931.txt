You are a senior full-stack engineer building a production-ready web app called:

AFROCAT CLUB PORTAL

GOAL
Build a modern responsive portal for a volleyball club that manages:
- Match stats & player performance reports
- Smart Focus training recommendations
- Attendance & discipline tracking
- Finance module (income/expenses, reports)
- Wellness & injury management (medical clearance)
- Awards, scouting, talent pipeline
- Role-based access control (RBAC): Admin, Manager, Coach, Statistician, Finance, Medical, Player

TECH STACK (use this exactly)
- Frontend: Vite + React + TypeScript + Tailwind
- Backend: Node.js + Express + TypeScript
- Database: SQLite via Prisma (so it runs free in Replit easily)
- Auth: email+password with JWT + bcrypt
- Validation: zod
- UI: clean dashboard with sidebar, tables, filters, export buttons
- Use a shared API client on the frontend (fetch wrapper)

PROJECT SETUP
1) Create a monorepo structure:
   /server (Express API)
   /client (React app)
2) Provide a single "npm run dev" root script that runs both server and client concurrently.
3) Add seed data so the app is usable immediately after run.

CORE ENTITIES (Prisma schema)
- User: id, fullName, email, passwordHash, role, teamId(optional), playerId(optional), createdAt
- Team: id, name, category (MEN|WOMEN|VETERANS|JUNIORS), season, createdAt
- Player: id, teamId, firstName, lastName, gender, dob, jerseyNo, phone, position, status (ACTIVE|SUSPENDED|INJURED), photoUrl(optional), createdAt
- Match: id, teamId, opponent, matchDate, venue, competition, result (W|L), setsFor, setsAgainst, notes(optional)
- PlayerMatchStat: id, matchId, playerId,
    spikesKill, spikesError,
    servesAce, servesError,
    blocksSolo, blocksAssist,
    receivePerfect, receiveError,
    digs, settingAssist, settingError,
    pointsTotal, minutesPlayed
- SmartFocus: id, playerId, matchId, focusAreas (JSON array), generatedAt
- AttendanceSession: id, teamId, sessionDate, sessionType (TRAINING|MATCH|GYM), notes(optional)
- AttendanceRecord: id, sessionId, playerId, status (PRESENT|LATE|ABSENT|EXCUSED), reason(optional)
- DisciplineCase: id, playerId, caseType, description, points, actionTaken, caseDate
- FinanceTxn: id, txnDate, type (INCOME|EXPENSE), category, amount, description, reference(optional), createdByUserId
- Injury: id, playerId, injuryType, severity (LOW|MEDIUM|HIGH), startDate, status (OPEN|CLEARED), clearanceNote(optional), clearedByUserId(optional)
- ScoutingReport: id, playerId(optional), prospectName(optional), notes, rating (1-10), createdAt
- Award: id, playerId, awardType (MVP|MOST_IMPROVED|BEST_SERVER|BEST_BLOCKER|COACH_AWARD), awardMonth, notes(optional)

BUSINESS RULES
A) RBAC
- Admin/Manager: everything
- Coach: teams, players, attendance, matches, stats view, smart focus view
- Statistician: create matches + enter stats + generate reports
- Finance: finance module only (plus read-only team/player lists)
- Medical: injury module only (plus read-only player lists)
- Player: can only view own profile, own stats history, smart focus, attendance summary, awards, injury status

B) Smart Focus Engine
After stats are submitted for a match, auto-generate Smart Focus for each player using simple rule logic:
- If servesError high => "Serving consistency"
- If spikesError high => "Attack control"
- If receiveError high => "Serve reception"
- If settingError high => "Setting accuracy"
- If blocks low and position is middle => "Blocking timing"
Store focusAreas as JSON array with priority ordering.

C) Attendance & Discipline
- If a player has >=3 ABSENT in last 30 days => auto-create a DisciplineCase "Attendance warning" with points.
- Show attendance % per player and per team.

D) Injury clearance
- If Injury status is OPEN => player status becomes INJURED and show as "Unavailable"
- Only Medical/Admin can set CLEARED.

E) Performance scoring
Compute an efficiency score for each PlayerMatchStat:
- Start at 0
- Add: spikesKill*2 + servesAce*2 + blocksSolo*2 + blocksAssist + digs + settingAssist
- Subtract: spikesError*2 + servesError*2 + receiveError*2 + settingError*2
Save computed score to pointsTotal.

FEATURES (must implement)
1) Auth pages (login/register) + role-based routing
2) Dashboard:
   - Admin view: stats cards (players, teams, matches, income/expenses, injuries open)
   - Coach view: next sessions, team attendance, player focus list
   - Player view: own performance trend + smart focus list
3) Teams CRUD (Admin/Manager)
4) Players CRUD with search/filter (Admin/Manager/Coach read, Player self-view)
5) Matches CRUD (Coach/Statistician)
6) Enter stats page:
   - pick match
   - table of players
   - input stats
   - submit => saves PlayerMatchStat + generates SmartFocus
7) Reports pages:
   - Match report (team summary + top performers)
   - Player report (history, scores, focus)
   - Attendance report (per team, per player)
   - Finance report (monthly summary)
   Provide “Export JSON” and “Print-friendly view” (PDF-ready HTML page).
8) Attendance module:
   - create session
   - mark attendance in a grid
9) Finance module:
   - add income/expense
   - categories
   - monthly totals
10) Injury module:
   - log injury
   - clearance workflow
11) Awards module:
   - assign awards
12) Seed data:
   - create at least 1 team and 8 players
   - create sample match, stats, finance txns, attendance sessions, injuries

API REQUIREMENTS
- REST endpoints grouped by module: /auth, /teams, /players, /matches, /stats, /attendance, /finance, /injuries, /awards, /reports
- Use zod validation on inputs
- Consistent error handling and auth middleware

QUALITY
- Clean UI, not cluttered, mobile friendly
- Use loading states, empty states, and toasts
- Add README with:
  - how to run
  - how to seed
  - demo login credentials for each role
  - project structure
  - common troubleshooting

DELIVERABLE
Generate the full working codebase now, including Prisma schema, migrations/seed, server routes, client pages, and styling.
