DATABASE / API CHANGES

1) Ensure API can fetch players with photos, jersey numbers, positions:
GET /teams/:teamId/players
Return player fields: id, fullName, jerseyNo, position, dob, photoUrl

2) Stats entry endpoints (bulk upsert)
POST /stats/match/:matchId
Body:
{
  matchMeta: { opponent, competition, venue, matchDate, matchTime, coachName },
  stats: [
    {
      playerId,
      spikesKill, spikesError,
      servesAce, servesError,
      blocksSolo, blocksAssist,
      receivePerfect, receiveError,
      digs,
      settingAssist, settingError,
      minutesPlayed
    }
  ]
}

Server actions on save:
- upsert PlayerMatchStat for each player
- compute efficiency score pointsTotal using current formula
- generate SmartFocus for each player (store focusAreas JSON)
- return a summary response:
  { matchId, updatedCount, topPerformers, teamTotals }

3) Coach dashboard aggregation endpoints
GET /dashboard/coach/summary?teamId=...
Return:
- last 5 matches summary
- team totals for selected match
- top 5 players by pointsTotal
- players with highest errors
- SmartFocus highlights (top 3 focus areas)

4) Player dashboard endpoints (self + coach view)
Player self:
GET /players/me/dashboard
Return:
- last 10 match stats
- performance trend (pointsTotal per match)
- SmartFocus history
- attendance summary
- injury status
- active contract summary (if present)

Coach viewing player:
GET /players/:playerId/dashboard
RBAC:
- Admin/Manager/Coach can view players in their tenant/team
- Player can only view their own
