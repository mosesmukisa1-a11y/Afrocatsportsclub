ELIGIBILITY RULES (must enforce)

A player is ELIGIBLE only if:
1) Player.status == ACTIVE
2) No OPEN Injury exists for the player (Injury.status != CLEARED)
3) PlayerContract (if contract system exists):
   - must have at least one ACTIVE contract OR contract rules allow no contract
   - if contract expired => NOT eligible
4) Player.eligibilityStatus != NOT_ELIGIBLE

On validation failure, return a reason list per player:
- "Player is INJURED (open injury)"
- "Player is SUSPENDED"
- "Contract expired"
- "Eligibility set to NOT_ELIGIBLE: <eligibilityNotes>"

BACKEND API

1) Fetch eligible players for team
GET /teams/:teamId/players/eligible?matchDate=YYYY-MM-DD
Returns:
- eligiblePlayers[]
- ineligiblePlayers[] with reasons

2) Create/Update match squad (Starting 12)
POST /matches/:matchId/squad
Body:
{
  playerIds: string[]   // must be exactly 12
}
Rules:
- Must be exactly 12 players
- Validate eligibility for all selected
- Save MatchSquad + MatchSquadEntry (upsert: replace old selection)

3) Get match squad
GET /matches/:matchId/squad
Returns selected players

4) Generate O-2bis must use MatchSquad if present
POST /o2bis/generate
Body:
{
  teamId,
  matchId,
  opponent, date, time, venue, competition
  // optional: playerIds override; if absent use saved squad
}
Rules:
- If MatchSquad exists -> use it
- Else require playerIds exactly 12
- Validate eligibility before generating
- Auto-fill coach and officials (see below)
Return { fileUrl, documentId }

OFFICIALS AUTO-FILL (no manual typing)
