  id          String @id @default(cuid())
  tenantId    String
  title       String?

MEDIA UPLOAD & TAGGING (CLIENT + API)

API
1) Public fetch (no auth):
GET /public/media?cursor=
- returns APPROVED + PUBLIC

GET /public/shop
- returns isPublic=true and isActive=true

2) Admin/Coach media management:
POST /media (upload image, title, caption, visibility)
- RBAC: Admin/Manager can upload
- Players can upload to PENDING_REVIEW only

POST /media/:id/approve   (Admin/Manager)
POST /media/:id/reject    (Admin/Manager)
GET /media/pending        (Admin/Manager/Coach optional)

3) Tagging:
POST /media/:id/tags
Body: { taggedPlayerIds:[], taggedUserIds:[] }
- RBAC: Admin/Manager/Coach can tag
- Players can request tags via:
POST /media/:id/tag-request
Body: { playerId }  // only self
Admin approves tag requests.

4) Dashboard media:
GET /players/me/media
- returns media posts where MediaTag.taggedPlayerId == playerId AND status=APPROVED

GET /coaches/me/media
- returns media posts where taggedUserId == coachUserId AND status=APPROVED

UI
- Admin: Media Manager page
  - Upload, approve/reject, set visibility
  - Tag people in photo (multi-select players/coaches)
- Player dashboard:
  - “My Media” section (photos they appear in)
  - “Upload My Sports Photo” button -> creates PENDING_REVIEW MediaPost with visibility TEAM_ONLY or PRIVATE default

ATTENDANCE SELF-CHECK (PLAYER) + COACH CONFIRM

UI
- Player sees upcoming training sessions
- For each session, a “Check In” button:

- Coach sees “Pending Confirmations” list:

  - confirmedByUserId set

API
POST /attendance/sessions/:id/checkin (PLAYER only)
- create/upsert AttendanceRecord with selfMarked=true

POST /attendance/records/:id/confirm (COACH/Admin)
Body: { status, notes }
- sets confirmedAt and confirmedByUserId
