UI UPGRADE: PLAYER DASHBOARD (DIGITAL FIFA-STYLE) WITH PHOTO + UPCOMING FIXTURE + EXISTING METRICS (AFROCAT COLORS)

GOAL
Redesign the Player Dashboard UI to match the attached reference layout:
- Left: Player Profile Card with photo + jersey + team + position + key totals
- Right: Performance chart panel + secondary chart panel + Statistics panel
- Add Upcoming Fixture card/section (next match) on the dashboard
- Keep all existing data already shown in the dashboard (stats history, smart focus, attendance, injury, reports)

STYLE (AFROCAT THEME)
Use Afrocat colors + premium dark UI:
- Background: deep charcoal/near-black
- Cards: dark slate with soft gradients
- Accent: Afrocat gold + white text
- Buttons: gold primary, subtle outline secondary
- Rounded corners, soft shadows, clean spacing
- Typography similar to reference (bold headings, small muted labels)

TECH
Client: React + TypeScript + Tailwind
Charts: Recharts (bar chart + line chart)
No external UI kit required; keep it sleek.

DATA REQUIREMENTS (API)
1) Player dashboard endpoint (already exists or create):
GET /players/me/dashboard
Return:
{
  player: { id, fullName, photoUrl, jerseyNo, position, team: { id, name }, totals: { matches, points, aces, blocks, digs }, status },
  performanceByMatch: [{ date, opponent, pointsTotal, result, isHome }],
  performanceTrend: [{ label, value }],            // pointsTotal by match date
  smartFocus: [{ matchDate, focusAreas }],
  attendance: { rate30d, absent30d, late30d },
  injury: { status, notes },
  reports: [{ type, date, url }]
}

2) Upcoming fixture endpoint:
GET /players/me/upcoming-fixture
Return:
{
  matchId,
  date,
  time,
  venue,
  competition,
  opponent,
  teamName,
  isHome,
  coachName
}
If no upcoming fixture: return null.

3) Coach can view same dashboard for a player:
GET /players/:playerId/dashboard
RBAC: Admin/Manager/Coach only, tenant-scoped.
