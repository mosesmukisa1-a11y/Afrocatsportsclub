ADD: PERMANENT ADMIN + ADMIN-ONLY PASSWORD RESET FLOW

GOAL
1) Keep the permanent admin provisioning via env vars (ensureAdmin script):
   ADMIN_EMAIL, ADMIN_PASSWORD, ADMIN_TENANT_SLUG
   - Upsert admin with bcrypt-hashed password
   - emailVerified=true, accountStatus=ACTIVE, role=ADMIN
   - Do NOT log the plain password anywhere

2) Add an ADMIN-ONLY password reset flow so users who forget passwords contact admin.
   - Users cannot self-reset
   - Only Admin can trigger reset and set a temporary password or generate a one-time reset link

SECURITY RULES (NON-NEGOTIABLE)
- Never store plain passwords.
- Never log temporary passwords in server logs.
- Never commit secrets to GitHub.
- Tenant scoped: Admin can only reset users within their tenant.
- Prevent IDOR: only ADMIN role can reset.
- Add rate limiting on reset endpoints.

DATABASE CHANGES (Prisma)

Extend User:
User {
  ...
  passwordResetTokenHash String?
  passwordResetTokenExp  DateTime?
  mustChangePassword     Boolean @default(false)
  lastPasswordResetAt    DateTime?
}

OPTIONAL Audit:
PasswordResetAudit {
  id          String @id @default(cuid())
  tenantId    String
  adminUserId String
  targetUserId String
  resetMethod ResetMethod
  createdAt   DateTime @default(now())
  notes       String?

  @@index([tenantId])
  @@index([adminUserId])
  @@index([targetUserId])
}

enum ResetMethod { TEMP_PASSWORD ONE_TIME_LINK }

BACKEND API (ADMIN ONLY)

1) Search users (for admin UI)
GET /admin/users?query=
Returns: id, fullName, email, role, accountStatus, emailVerified

2) Reset method A: Set Temporary Password (recommended for simplicity)
POST /admin/users/:userId/reset-password
Body:
{
  method: "TEMP_PASSWORD",
  tempPassword: string
}
Server:
- validate ADMIN
- validate user is same tenantId
- bcrypt hash tempPassword -> passwordHash
- set mustChangePassword=true
- clear any existing reset token
- set lastPasswordResetAt=now
- create PasswordResetAudit record
Return: { ok: true }
IMPORTANT: do NOT log tempPassword.

3) Reset method B: Generate one-time reset link (more secure)
POST /admin/users/:userId/reset-password-link
Body:
{
  method: "ONE_TIME_LINK"
}
Server:
- generate random token 32+ bytes
- store hash(token) in passwordResetTokenHash
- set exp = now + 1 hour
- set mustChangePassword=true
- create PasswordResetAudit record
Return: { resetLink } where resetLink = FRONTEND_URL + "/reset-password?token=..." + "&email=..."
If SMTP is configured, send email to user with link; otherwise show link ONLY in admin UI (not logs).

4) Complete reset (public, token-based)
POST /auth/reset-password
Body: { email, token, newPassword }
Server:
- find user by email + tenantId (from token match)
- verify token hash matches and not expired
- bcrypt newPassword -> passwordHash
- clear reset token fields
- set mustChangePassword=false
- set lastPasswordResetAt=now
Return: success
